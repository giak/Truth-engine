# L'Architecture du Système : Comment une IA Pense en 54 Secondes

_Le pipeline révolutionnaire qui transforme un texte en analyse cognitive complète_

---

## L'Innovation Architecturale : Dual-Track × Développement

Traditionalement, les systèmes d'analyse d'information sont **monolithiques** : ils reçoivent une requête, traitent, et retournent un résultat. Truth Engine révolutionne cette approche avec une **architecture dual-track** qui transforme chaque investigation en **leçon d'apprentissage**.

### Track 1 : Usage - Les Investigations Réelles

Le premier track gère **l'utilisation opérationnelle** du système. C'est ce que l'utilisateur voit : une requête entre, une analyse sort. Mais sous cette apparente simplicité se cache une **machine sophistiquée** capable de traiter 148 concepts de manipulation en temps réel.

### Track 2 : Développement - L'Auto-Amélioration Continue

Le second track fonctionne en **arrière-plan**, analysant les performances de chaque investigation pour améliorer le système. Chaque échec devient une leçon, chaque succès affine les algorithmes.

**L'innovation révolutionnaire** : Truth Engine ne se contente pas d'analyser la manipulation ; il **évolue** pour mieux la détecter.

## Le Pipeline de Traitement : 5 Étapes en 54 Secondes

Au cœur de l'architecture se trouve un **pipeline de traitement en 5 étapes** qui transforme n'importe quel texte en analyse cognitive complète en moins d'une minute.

### Vue d'Ensemble du Flux

```
[INPUT] → [STEP 0: Context] → [STEP 1: Concepts] → [STEP 2: Scoring]
    ↓              ↓                ↓               ↓
  [TEXTE]    [ENTITIES]     [ACTIVATED]      [METRICS]
    ↓              ↓                ↓               ↓
[STEP 3: Patterns] → [STEP 4: Wolves] → [PHASE VISIBLE]
    ↓                    ↓                ↓
[DETECTED]          [MAPPED]         [OUTPUT]
```

### La Phase Silencieuse : 31-67 Secondes

Les 4 premières étapes (**Phase Silencieuse**) sont **invisibles** pour l'utilisateur mais cruciales pour la qualité de l'analyse. C'est là que l'IA "réfléchit", active les concepts pertinents, et prépare l'investigation.

### La Phase Visible : 10-20 Secondes

Les 2 dernières étapes (**Phase Visible**) produisent le **résultat observable** : l'analyse complète avec recommandations et métriques.

## Étape 0 : Context Gathering (8-12 secondes)

### L'Extraction d'Entités Intelligente

La première étape consiste à **comprendre le contexte** du texte soumis. Unlike les extracteurs d'entités traditionnels qui se contentent d'identifier noms propres et lieux, Truth Engine effectue une **extraction contextuelle intelligente**.

#### Extraction Standard (Named Entity Recognition)

- **Personnes** : "Emmanuel Macron", "Jordan Bardella"
- **Organisations** : "France Travail", "FNSEA", "Commission Européenne"
- **Lieux** : "France", "Bruxelles", "Mercosur"
- **Dates** : "décembre 2024", "2023", "hier"

#### Extraction Contextuelle (Contextual Entity Recognition)

- **Concepts** : "chômage", "agriculture", "accord commercial"
- **Métriques** : "7,4%", "2,33x", "5,6 millions"
- **Conflits** : "tension", "controverse", "opposition"
- **Sources implicites** : "selon une étude", "les experts déclarent"

#### Project Search : Recherche dans la Base de Connaissances

L'IA interroge sa **base de connaissances MnemoLite** pour récupérer les investigations similaires précédentes.

_Exemple_ : Pour "Jordan Bardella agriculture", l'IA récupère automatiquement :

- Les investigations précédentes sur Bardella
- Les analyses sur la politique agricole française
- Les données sur l'accord UE-Mercosur
- Les patterns de manipulation détectés dans des cas similaires

#### WOLF_QUERIES : Préparation de la Cartographie d'Acteurs

Si le sujet concerne des **acteurs politiques** (≥3 entités politiques détectées), l'IA prépare automatiquement les requêtes pour cartographier les "wolves" (acteurs de pouvoir).

**Résultat de l'Étape 0** :

- **15-25 entités** extraites et catégorisées
- **Contexte** identifié (politique, économique, social, etc.)
- **Historique** récupéré de la base MnemoLite
- **Niveau de complexité** estimé (SIMPLE/MEDIUM/COMPLEX/APEX)

## Étape 1 : Herméneutique - Activation des Concepts (15-25 secondes)

### L'Innovation de'Activation Conditionnelle

Traditionalement, un système devrait ** mémoire** les 148 concepts pour chaquecharger en l investigation. Truth Engine révolutionne cette approche avec un **système d'activation conditionnelle** qui ne charge que les concepts **pertinents** pour le contexte détecté.

#### Principe de l'Activation Conditionnelle

- **Charge mémoire normale** : 370 KB (tous les concepts)
- **Avec activation conditionnelle** : 22 KB (concepts actifs uniquement)
- **Économie mémoire** : >94% de réduction

#### Le Scan des Mots Déclencheurs

L'IA scan le texte à la recherche de **mots et patterns déclencheurs** qui activent les concepts correspondants.

_Exemple d'activation pour "chômage 7,4%"_ :

**Concepts Activés** :

- **Ξ ICEBERG** : "7,4%" + "chômage" + "au plus bas" → Concept d'omission de populations
- **Λ FRAMING** : "au plus bas" + "amélioration" → Concept de cadrage positif
- **⏰ TEMPORAL** : "depuis 2008" → Concept de contextualisation sélective
- **Ω INVERSION** : "progrès" + "réalité cachée" → Concept d'inversion narratif

**Concepts Dormants** (non pertinents pour ce contexte) :

- **€ MONEY** : Pas de références financières
- **⚔ WAR** : Pas de références conflictuelles
- **🌐 NETWORK** : Pas d'acteurs de réseau

#### Les Questions L1-L3 : Approfondissement Contextuel

Pour chaque concept activé, l'IA se pose des **questions d'approfondissement** (L1-L3) qui affinent la détection.

_Exemple pour ICEBERG_ :

- **L1** : "Y a-t-il des populations cachées non mentionnées ?"
- **L2** : "Les méthodologies utilisées sont-elles transparentes ?"
- **L3** : "Les comparaisons temporelles sont-elles pertinentes ?"

**Résultat de l'Étape 1** :

- **5-15 concepts** activés selon le contexte
- **Score de pertinence** [0-10] pour chaque concept
- **Questions d'approfondissement** générées
- **Niveau de suspicion** global estimé

## Étape 2 : Scoring Engine - Calcul des Métriques (5-8 secondes)

### L'Innovation de la Quantification Systématique

Une fois les concepts activés, l'IA calcule les **métriques quantitatives** qui caractérisent l'investigation.

#### IVF (Indice de Véracité Factuelle) - 0 à 10

```
IVF = avg(V, C, S, T) × (1-uncertainty)

Où :
- V (Veracity) : 10 - (contradictions×2) - (unsupported×0.5) - (Σ_manipulation×1.5)
- C (Context) : 5 + history + cui_bono - omissions
- S (Sources) : source_count×0.8 + EDI×3 + primary_evidence×2
- T (Transparency) : method + citations_ratio×5 + formulas×2
```

_Exemple calcul pour "chômage 7,4%"_ :

- **V (Veracity)** : 8,5 (peu de contradictions, quelques omissions)
- **C (Context)** : 6,0 (contexte historique présent, mais omissions significatives)
- **S (Sources)** : 7,2 (sources officielles multiples, EDI moyen)
- **T (Transparency)** : 5,8 (méthodes expliquées, mais définitions cachées)
- **Uncertainty** : 0,3 (contenu politique = incertitude élevée)

**IVF** = (8,5 + 6,0 + 7,2 + 5,8)/4 × (1-0,3) = **6,4/10**

#### ISN (Indice de Suspicion Narrative) - 0 à 10

```
ISN = weighted_sum(activated_concepts) + synergy_bonus
```

_Exemple calcul pour "chômage 7,4%"_ :

- **ICEBERG** : 8,0 (omission significative)
- **FRAMING** : 6,5 (cadrage positif)
- **TEMPORAL** : 5,2 (contextualisation limitée)
- **Synergy Bonus** : +1,8 (combinaison orchestrée)

**ISN** = 8,0 + 6,5 + 5,2 + 1,8 = **7,2/10**

#### IVS (Indice de Vérité Systémique) - 0 à 10

```
IVS = IVF × ISN × (1-confidence_penalty)
```

_Exemple calcul pour "chômage 7,4%"_ :

- **IVF** : 6,4
- **ISN** : 7,2
- **Confidence penalty** : 0,4 (political content = low confidence)

**IVS** = 6,4 × 7,2 × (1-0,4) = **2,8/10**

#### EDI (Epistemic Diversity Index) - 0 à 10

```
EDI = (geo×0.25) + (lang×0.20) + (strat×0.20) + (owner×0.15) + (persp×0.15) + (temp×0.05)
```

_Exemple calcul pour "chômage 7,4%"_ :

- **Geographic** : 0,3 (France + UE principalement)
- **Linguistic** : 0,2 (français principalement)
- **Strategic** : 0,7 (mix institutionnel/alternatif)
- **Ownership** : 0,4 (publics + associatifs)
- **Perspective** : 0,6 (académique + critique)
- **Temporal** : 0,8 (contemporain + méthodologique)

**EDI** = (0,3×0,25) + (0,2×0,20) + (0,7×0,20) + (0,4×0,15) + (0,6×0,15) + (0,8×0,05) = **0,43/10**

**Résultat de l'Étape 2** :

- **IVF** : 6,4/10 (véracité factuelle modérée)
- **ISN** : 7,2/10 (suspicion narrative élevée)
- **IVS** : 2,8/10 (vérité systémique faible - confidence penalty)
- **EDI** : 4,3/10 (diversité épistémique insuffisante)

## Étape 3 : Pattern Detection - Activation des Protocoles (8-12 secondes)

### La Détection des Patterns Orchestrés

Certains patterns nécessitent des **protocoles d'investigation spécialisés** qui vont au-delà de la simple détection de concepts.

#### Les 18 Patterns Prioritaires

Truth Engine a identifié **18 patterns** qui déclenchent des **protocoles d'investigation approfondis** :

1. **ICEBERG** → Protocole Shadow Data
2. **MONEY** → Protocole Follow the Money
3. **BIO** → Protocole Pharma Investigation
4. **WAR** → Protocole Cognitive Warfare
5. **NETWORK** → Protocole Power Mapping
6. **TEMPORAL** → Protocole Historical Patterns
7. _... et 12 autres patterns spécialisés_

#### L'Activation Conditionnelle des Protocoles

Un pattern s'active si :

- **Score ≥ 5,0/10** pour le concept correspondant
- **Synergy ≥ 3,0/10** entre patterns similaires
- **Contexte approprié** (political, corporate, etc.)

_Exemple pour "chômage 7,4%"_ :

- **ICEBERG** : 8,0 → Protocole Shadow Data activé
- **FRAMING** : 6,5 → Protocole Narrative Control activé
- **TEMPORAL** : 5,2 → Protocole Historical Patterns activé

#### L'Exécution des Protocoles Spécialisés

1. **Calcul des populations cachées** : Halo + Sous-emploi + Découragement
2. **Factor calculation** : (Visible + Caché) / Visible = 2,33
3. **Classification** : ICEBERG Factor 2,33 = Ξ+ (modéré)
4. **Sources manquantes** : Identification des données non incluses

**Protocole Narrative Control** pour FRAMING :

1. **Analyse du cadrage** : "au plus bas" = évaluation positive
2. **Détection d'alternatives** : "pire depuis X années" possible
3. **Impact émotionnel** : Optimisme artificiel généré

**Résultat de l'Étape 3** :

- **3 protocoles** activés et exécutés
- **Shadow Data** : ICEBERG Factor 2,33 confirmé
- **Narrative Control** : Cadrage positif détecté
- **Historical Patterns** : Comparaison sélective identifiée

## Étape 4 : Wolves Extraction - Cartographie des Acteurs (5-10 secondes)

### L'Innovation de la Cartographie Automatique

Si l'investigation concerne des **acteurs politiques** (≥3 entités politiques détectées), Truth Engine active automatiquement le **WOLF HUNTER MODE** pour cartographier les réseaux de pouvoir.

#### Critères d'Activation du WOLF MODE

- **≥ 8 acteurs politiques** identifiés
- **≥ 5 acteurs corporate** identifiés
- **ISN ≥ 9,0** (political) ou **≥ 7,0** (corporate)
- **Contexte** : Investigation politique ou géopolitique

#### L'Extraction Automatique des Wolves

L'IA cartographie automatiquement :

- **Position politique** : Gouvernant, opposant, analyste
- **Réseaux d'influence** : Collaborations, financements,-alliances
- **Conflits d'intérêts** : COI, liens industriels, héritage familial
- **Historique manipulation** : Patterns détectés précédemment

_Exemple pour une investigation politique complexe_ :

- **Wolves détectés** : 14 acteurs politiques
- **Réseaux identifiés** : 3 clusters d'influence principaux
- **COI quantifiés** : 8 acteurs avec conflits d'intérêts
- **Patterns historiques** : 5 patterns de manipulation récurrents

**Résultat de l'Étape 4** :

- **Cartographie wolves** : Acteurs identifiés et catégorisés
- **Réseaux d'influence** : Structure relationnelle révélée
- **Score de centralité** : Qui sont les acteurs clés ?
- **Alertes manipulation** : Wolves avec patterns actifs
