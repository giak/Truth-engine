# 🔍 CLUSTER INVESTIGATION PROTOCOLS - General Procedures for All Clusters

_Common investigation protocols to enhance depth and consistency across all cluster types_

## PURPOSE

Provide standardized investigation procedures that can be activated when any cluster concept scores ≥5. These protocols ensure that each activated cluster triggers a structured deep dive rather than just surface-level analysis.

## ACTIVATION CRITERIA

- Triggered when **any cluster concept score ≥5**
- Applies to all cluster types: ICEBERG, MONEY, NETWORK, FRAMING, INVERSION, etc.
- Can be overridden by cluster-specific protocols (e.g., ICEBERG's shadow data protocols)

## GENERAL INVESTIGATION PROTOCOL (L1-L3 Depth)

### PHASE 1: PRELIMINARY VERIFICATION

```yaml
FOR each activated_concept WITH score ≥5:
  → STEP 1: Validate concept relevance with direct quotes from source material
  → STEP 2: Cross-verify with at least 1 additional source
  → STEP 3: Identify counterarguments or alternative interpretations
  → STEP 4: Document confidence level (Low/Medium/High)
  → STEP 5: Assign investigation depth level (L1-L3)
```

### PHASE 2: SOURCE EXPANSION

```yaml
FOR each activated_concept WITH score ≥6:
  → SEARCH 1: Primary sources related to the concept
  → SEARCH 2: Academic or expert analysis of the pattern
  → SEARCH 3: Historical examples of similar patterns
  → SEARCH 4: Counter-perspectives or debunking attempts
  → EVALUATE: Source quality using EDI criteria
```

### PHASE 3: PATTERN ANALYSIS

```yaml
FOR each activated_concept WITH score ≥7:
  → ANALYZE: Pattern manifestation in current context
  → COMPARE: With historical precedents
  → IDENTIFY: Root causes and enabling factors
  → ASSESS: Impact on stakeholders
  → ESTIMATE: Pattern strength and persistence
```

## INVESTIGATION DEPTH INDICATORS

### LEVEL L0 (Surface)

- Basic concept recognition
- Single source validation
- Minimal analysis

### LEVEL L1 (Basic)

- Concept identification with quotes
- Cross-verification with 1 additional source
- Simple interpretation

### LEVEL L2 (Intermediate)

- Multiple source validation
- Counterarguments considered
- Pattern manifestation described

### LEVEL L3 (Advanced)

- Pattern analysis with historical context
- Impact assessment
- Source quality evaluation

### LEVEL L4 (Deep)

- Hypothesis generation
- Multi-angle investigation
- Quantitative analysis

### LEVEL L5 (Very Deep)

- Network mapping
- Causal chain analysis
- Expert consultation

### LEVEL L6 (Comprehensive)

- Full investigation tree activation
- Parallel branch exploration
- Deep dive reporting

## QUALITY CONTROL CHECKPOINTS

### AFTER CONCEPT ACTIVATION:

1. All activated concepts have direct source quotes
2. Each concept has at least 2 validation sources
3. Counterarguments are documented
4. Confidence levels are assigned

### AFTER SOURCE EXPANSION:

1. ≥1 primary source per high-score concept
2. Academic/expert analysis included
3. Historical precedents identified
4. Source quality meets EDI targets

### AFTER PATTERN ANALYSIS:

1. Pattern impact is quantified
2. Root causes are identified
3. Future implications are discussed
4. Investigation depth level is validated

## OUTPUT REQUIREMENTS

### CONCEPT ANALYSIS:

- Concept symbol + name
- Score with justification
- Triggering quote from source
- Technique name (DSL term)
- Hidden implication
- Depth level (L0-L6)
- Confidence level (Low/Medium/High)
- Validation sources

### PATTERN SUMMARY:

- Pattern manifestation in context
- Historical precedents
- Impact on stakeholders
- Counterarguments
- Investigation recommendations

## METRICS FOR SUCCESS

- **Depth Coverage**: ≥80% of activated concepts at L2+
- **Source Quality**: ≥70% of sources at ◉ or ◈
- **Confidence Level**: Average confidence ≥Medium
- **Counterarguments**: ≥1 per high-score concept
- **Validation Sources**: ≥2 per concept
