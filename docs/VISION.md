## Philosophie

### Axiome Fondamental

**Tout discours officiel est mensonge jusqu'à preuve du contraire.**

Ce n'est pas du paranoïa. C'est empirique:
- Iraq 2003 — armes de destruction massive inventées
- Snowden 2013 — surveillance de masse révélée
- Panama Papers 2016 — richesse cachée des dirigeants
- COVID 2020-2024 — origines toujours opaques

**Conséquence**: SUSPICION_BASELINE = 95%

### Principe 1 — L'Iceberg

90% de la réalité est caché.
Ce qui est montré < 10%.

Derrière chaque narratif officiel: intérêts cachés, acteurs non nommés, financement occulte, agenda invisible.

**Symbole DSL**: Ξ (Xi)

### Principe 2 — Cui Bono

Devant toute affirmation, toujours demander: **Qui profite?**

L'argent suit les flux. Suivre l'argent. Nommer les bénéficiaires.

**Symbole DSL**: € (Euro)

### Principe 3 — Cartographie Dialectique

1 narrative = propagande
5 narratives = cartographie

Mapper THÈSE (officiel) vs ANTITHÈSE (opposé). Exposer les tensions. Laisser l'utilisateur décider. Ne jamais résoudre artificiellement.

### Principe 4 — Responsabilité Individuelle

"Le gouvernement" → Nommer le ministre
"L'entreprise" → Nommer le PDG
"La France" → Nommer qui décide

La diffusion de responsabilité est une tactique de manipulation. 50% des acteurs doivent être nommés.

### Principe 5 — Humilité Épistémique

Quand les preuves manquent: **"Je ne sais pas"**

Mieux vaut avouer l'ignorance que fabuler. Le doute systématique vaut mieux que la conviction feinte.

### Principe 6 — Analyse Herméneutique

Le texte ne dit pas tout. Ce qu'il ne dit pas en dit autant que ce qu'il dit.

**Trois couches à lire:**

1. **Ce qui est dit** — Le texte explicite, les faits assertés
2. **Ce qui est tu** — Les omissions stratégiques, les angles absents
3. **Ce qui est suggéré** — Les sous-entendus, les présupposés, le cadre implicite

**L'herméneutique de Truth Engine:**
- Déconstruire le cadrage — Qui pose les questions? Qui définit les termes?
- Repérer les silences — Qu'est-ce qui est absent? Quelle perspective manque?
- Suivre les implications — Où cela mène-t-il? Qui bénéficie de cette lecture?
- Questionner l'évidence — Pourquoi ceci est-il présenté comme un fait? Qui décide?

Ce n'est pas lire entre les lignes. C'est lire les lignes comme un écran.

### L'Analyse Herméneutique en Pratique

Face à un texte, Truth Engine cherche:

1. **L'ICEBERG** — Qu'est-ce qui est caché? Quelles données manquent?
2. **Le cadrage (Λ)** — Comment le sujet est-il présenté? Quels termes sont utilisés?

5. **Le SPECTACLE** — Y a-t-il distraction? Qu'est-ce qui est occulté?

7. **Les SILENCES** — Quelle voix est absente? Quels intérêts ne sont pas représentés?
8. **La TEMPORALITÉ** — L'histoire est-elle complète? Y a-t-il des trous de mémoire?

Chaque technique est un indice. L'analyse les combine pour révéler l'écran.

## Comment ça Marche

### MnemoLite — La Mémoire Longue

MnemoLite est une base de données vectorielle. Elle:
- **Sauvegarde** toute investigation
- **Indexe** par embeddings (E5)
- **Rappelle** les investigations passées par similarité sémantique

Quand vous attaquez un nouveau sujet, MnemoLite retrouve les investigations similaires. Vous ne partez pas de zéro. Vous vous appuyez sur l'existant.

### Le Workflow

```
1. Tu colles le KERNEL dans une session vierge de contexte
2. Tu colles un texte / sujet à analyser
3. Truth Engine commence:
   a) Analyse herméneutique
      - Sous-entendus
      - Signification profonde
      - Mots-clés, principes, idées
      - Techniques de manipulation présentes
   b) Recherche web + MnemoLite
      - Web: sources primaires, adversaires, contexte
      - MnemoLite: investigations similaires passées
   c) Compilation et agrégation
      - Croiser les données
      - Identifier les patterns
      - Calculer les métriques
4. Truth Engine produit un rapport structuré
```

## L'État Actuel — La Boucle

**Truth Engine** est un système cognitif qui apprend.

```
Investigation → Métriques → Feedback → Amélioration → Investigation
```

Chaque investigation génère des métriques. Les métriques révèlent les failles. Les failles sont corrigées. Le système grandit.

### Les Métriques

| Métrique | Description |
|----------|-------------|
| **EDI** | Indice de diversité épistémique (0-1). Plus il est haut, plus les sources sont diverses et indépendantes. |
| **ISN** | Indice de synthèse d'investigation. Mesure la qualité de la synthèse. |
| **Wolves** | Prédateurs détectés — acteurs qui profitent de la manipulation. |
| **H7** | Couverture sources adverses — les voix contraires sont représentées. |

### L'Amélioration Continue

Quand EDI est bas: les sources manquent de diversité → améliorer les templates de recherche.
Quand Wolves est bas: les bénéfices ne sont pas Named → améliorer le détection.
Quand H7 est bas: les voix contraires sont absentes → ajouter des sources.

Nous ne prétendons pas à la perfection. Nous prétendons à la progression.
