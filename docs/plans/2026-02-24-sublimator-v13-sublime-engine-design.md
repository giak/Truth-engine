# DESIGN DOC : SUBLIMATOR v13.0 — THE SUBLIME ENGINE

**Date** : 2026-02-24
**Status** : In Progress
**Topic** : Refining the "Omniscient" engine to reach "Magnum Opus" quality by purging technical artifacts and enhancing narrative capture.

## 1. Problem Statement
The v12.0 article was forensic-heavy but technically noisy. Labels like "DOMINO", "TRUTH ENGINE v12.0", and visible source tags `(◈ R1)` break the reader's immersion and dampen the narrative impact.

## 2. Proposed Solution : The Sublime Engine
Maintain the **Chain of Causality** logic from v12.0 but implement a **"Stealth Forensic"** layer. Validated facts must feel like an organic part of the story, not a line in a spreadsheet.

## 3. Design Components

### 3.1 Stealth Forensics (Invisible Authority)
- **Purge Meta-References** : No mentions of the tool, version, or "Audit" in the body.
- **Anonymous Citations** : Replace `(◈ R1)` with subtle footnotes `[1]` or integrated mentions ("Comme le confirment les registres de la Cour des Comptes...").
- **Clean Titles** : Evocative headings only. No "DOMINO" prefixes.

### 3.2 Narrative Tempo & Visual Impact
- **Bold Anchoring** : Keywords (names, dates, sums) must be **bolded** to create visual weight and facilitate scanning.
- **Breathing Space** : Frequent horizontal rules `---` and short, punchy paragraphs.
- **Capture Logic** : Every section must end with a "Cliffhanger" or a "Hard Truth" that pulls the reader into the next section.

### 3.3 Output Format (Magnum Opus v5)
- **Title** : Evocative, cinematic.
- **Intro** : High tension, no meta-talk.
- **Body** : Continuous causal flow with bolded facts.
- **Footer** : Comprehensive "Appendice Forensique" with sources, matching the footnote numbers.

## 4. Success Criteria
- [ ] No technical labels in titles or prose.
- [ ] Visual scanability (bold text usage).
- [ ] Length and depth matching v12.0 but with "literary" elegance.
- [ ] Total data coverage (no omission).
