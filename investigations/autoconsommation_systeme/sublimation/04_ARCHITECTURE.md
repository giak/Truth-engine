L'ordre A→B→C fonctionne comme un tout :

- **Acte I (A)** dévoile la capture. Le lecteur voit *qui* prend.

A sans B = la capture sans les conséquences. B sans C = l'effondrement sans le mécanisme. C sans A = l'aveuglement sans le moteur. Ensemble = auto-consommation.

## ARCHITECTURE LOGIQUE : POURQUOI A→C→B

La relation canonique établie par la dialectique est C→A→B (l'inversion comme condition de la capture, la capture comme moteur de l'effondrement). Mais l'architecture de lecture suit A→C→B pour une raison de révélation progressive :

| Ordre | Acte | Ce que le lecteur DÉCOUVRE | Ce que ça révèle sur le suivant |
|-------|------|---------------------------|-------------------------------|
| 1er | A — CAPTURE | La machine financière, les réseaux, l'auto-sabordage | Comment ça tient debout malgré la visibilité ? → C |
| 2e | C — INVERSION | Le pathogène-médecin, la sélection sombre, l'aveuglement | Que produit physiquement cette machine invisible ? → B |
| 3e | B — SYSTÈME | L'effondrement comme carburant, la biosphère comme combustible | Les trois ne font qu'un → VERDICT |

Chaque acte révèle quelque chose que le suivant rend incontournable :
- A révèle la capture → mais le lecteur peut penser « c'est un dysfonctionnement réformable »
- C révèle l'inversion → le lecteur comprend que la capture n'est pas réformable parce que le système ne peut pas se voir
- B révèle l'alimentation → le lecteur voit que l'effondrement n'est pas un coût mais le carburant

A sans C = réformisme naïf. C sans B = paranoïdia. B sans A = éco-anxiété sans mécanisme. Ensemble = auto-consommation.
